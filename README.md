# 간호 지식 저장소 (RN Knowledge Vault)

종합병원 간호 업무에 활용하기 위한 개인 지식 저장소입니다.
**Claude Code**(수집·정리) → **GitHub**(저장·이력) → **Obsidian**(열람·시각화) 흐름으로 운영합니다.

## ⚠️ 사용 원칙 (꼭 지킬 것)

1. **이 저장소는 Private** — 공개하지 않습니다.
2. **환자 정보(PHI) 절대 금지** — 환자 이름·차트번호·생년월일 등 개인식별정보는 어떤 노트에도 넣지 않습니다. 이곳은 "지식 사전"이지 "환자 기록"이 아닙니다.
3. **모든 지식은 참고용** — 최종 판단은 처방·원내 지침·최신 원문을 따릅니다. 각 노트 하단에 면책 문구를 둡니다.

## 폴더 구조

지식 영역별로 최상위 폴더를 나눕니다. **폴더는 얕게(2~3단계), 세부 분류는 태그/속성(YAML)으로** 하는 것이 이 저장소의 원칙입니다.

| 폴더 | 내용 |
|------|------|
| `업무/` | 내 실제 업무 정리 (인수인계, 프로토콜, 체크리스트 등) |
| `간호학/` | 간호학 지식 |
| `약학/` | 국내 일반의약품·전문의약품 정보 |
| `의학/` | 의학 지식 |
| `_템플릿/` | 새 노트를 만들 때 쓰는 Obsidian 템플릿 |

## 왜 폴더는 얕게, 분류는 태그로?

하나의 항목이 여러 분류에 걸치는 경우가 많기 때문입니다. (예: 이부프로펜 = 해열제 + 진통제 + 소염제)
폴더는 하나에만 넣을 수 있지만, 태그/속성은 여러 개를 동시에 달 수 있어 검색이 자유롭습니다.

```markdown
---
효능군: [해열제, 진통제, 소염제]
---
```

Obsidian에서 `효능군: 해열제` 같은 속성으로 검색하면 폴더 위치와 상관없이 한 번에 모입니다.

## 노트 작성 규칙

- 파일명은 항목 이름 그대로 (예: `타이레놀정500mg.md`)
- 상단에 YAML 속성(frontmatter)으로 분류 정보를 담습니다.
- 항목 간 관계는 `[[위키링크]]`로 연결합니다. (예: 병용금기 약을 링크로 걸면 그래프 뷰에 관계가 그려집니다.)
- 새 노트는 `_템플릿/`의 템플릿을 복사해서 시작합니다.
